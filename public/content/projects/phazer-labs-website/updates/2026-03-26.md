# Phazer Labs Go-Live!
Phazer Labs now has a public-facing home. The site communicates what we build — immersive festival installations, real-time visuals, AI-driven creative pipelines, and 3D fabrication — with the same signal-to-noise ratio we bring to every project. Deployed via GitHub Pages, built in collaboration with Claude as a creative-technical co-pilot.

## How It Was Built

### 01 — Direction & structure with Claude
Defined site architecture, tone, and content hierarchy in a collaborative session. Claude helped structure the narrative — what Phazer Labs does, who it's for, and how to present the work without fluff.

**Tools:** claude.ai · architecture · copywriting

### 02 — Design & code generation
Claude generated the full site codebase — semantic HTML, responsive CSS, and vanilla JS — following the visual direction for a dark, technical aesthetic consistent with the Phazer brand. Iterated in real-time via conversation.

**Tools:** claude.ai · HTML/CSS · responsive design

### 03 — Repo setup & GitHub Pages deploy
Pushed the codebase to a public GitHub repository and enabled Pages deployment from the main branch. Zero-config hosting with automatic deploys on push — no build step, no CI overhead.

**Tools:** GitHub · Pages · Git

### 04 — Review, polish & go-live
Final content review, mobile QA across breakpoints, and DNS / custom domain wiring. Site went live with no downtime and a clean Lighthouse score.

**Tools:** QA · responsive · DNS
